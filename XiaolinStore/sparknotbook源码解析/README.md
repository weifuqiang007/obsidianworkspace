## 开发规范

### 代码组织
- 按功能模块划分代码结构
- 公共代码提取到 `common` 目录
- 遵循 DRY 原则

### 文档规范
- 每个模块都有独立的文档
- 包含源码实现和业务逻辑
- 建立模块间的关联关系

### 架构原则
- 高内聚低耦合
- 分层设计（API层、Service层、Model层）
- 接口优先设计
