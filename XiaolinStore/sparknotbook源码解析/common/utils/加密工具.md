# 加密工具模块

## 概述

加密工具模块为 SparkNoteAI 应用提供 API 密钥的加密和解密功能，确保敏感信息安全存储。基于 Fernet 对称加密算法，实现了安全的密钥管理机制。

## 核心功能

### 1. 密钥管理

#### get_encryption_key()
```python
def get_encryption_key() -> bytes:
    """
    获取加密密钥

    从环境变量读取 ENCRYPTION_KEY
    如果没有设置，则生成一个新的密钥（仅用于开发环境）

    Returns:
        bytes: 32 字节的加密密钥
    """
```
**源码实现**:
```python
def get_encryption_key() -> bytes:
    """
    获取加密密钥

    从环境变量读取 ENCRYPTION_KEY
    如果没有设置，则生成一个新的密钥（仅用于开发环境）

    Returns:
        bytes: 32 字节的加密密钥
    """
    key_str = settings.ENCRYPTION_KEY

    if not key_str:
        # 开发环境：如果没有配置，生成一个新密钥
        # 生产环境必须配置 ENCRYPTION_KEY 环境变量
        key = Fernet.generate_key()
        logger.warning("未配置 ENCRYPTION_KEY，使用临时生成的密钥。请将此密钥添加到环境配置中")
        return key

    # 如果是字符串，转换为 bytes
    if isinstance(key_str, str):
        key_bytes = key_str.encode()
    else:
        key_bytes = key_str

    # 验证密钥格式
    try:
        Fernet(key_bytes)
    except ValueError:
        raise ValueError(
            "ENCRYPTION_KEY 格式无效。请使用 Fernet 生成的 32 字节 base64 编码密钥。\n"
            f"生成新密钥：python -c 'from cryptography.fernet import Fernet; print(Fernet.generate_key().decode())'"
        )

    return key_bytes
```

- **用途**: 获取 Fernet 加密所需的密钥
- **优先级**: 环境变量 > 开发环境自动生成
- **安全特性**:
  - 生产环境必须配置环境变量
  - 密钥格式验证
  - 自动类型转换

### 2. API 密钥加密

#### encrypt_api_key()
```python
def encrypt_api_key(api_key: str) -> str:
    """
    加密 API Key

    Args:
        api_key: 原始 API Key 字符串

    Returns:
        str: 加密后的字符串（base64 编码）
    """
```
**源码实现**:
```python
def encrypt_api_key(api_key: str) -> str:
    """
    加密 API Key

    Args:
        api_key: 原始 API Key 字符串

    Returns:
        str: 加密后的字符串（base64 编码）
    """
    f = Fernet(get_encryption_key())
    encrypted = f.encrypt(api_key.encode())
    return encrypted.decode()
```

- **用途**: 加密敏感的 API 密钥
- **实现**: 使用 Fernet 对称加密
- **输入**: 原始 API Key 字符串
- **输出**: base64 编码的加密字符串

### 3. API 密钥解密

#### decrypt_api_key()

```python
def decrypt_api_key(encrypted_key: str) -> str:
    """
    解密 API Key

    Args:
        encrypted_key: 加密的 API Key 字符串

    Returns:
        str: 解密后的原始 API Key
    """
```
**源码实现**:
```python
def decrypt_api_key(encrypted_key: str) -> str:
    """
    解密 API Key

    Args:
        encrypted_key: 加密的 API Key 字符串

    Returns:
        str: 解密后的原始 API Key
    """
    f = Fernet(get_encryption_key())
    decrypted = f.decrypt(encrypted_key.encode())
    return decrypted.decode()
```

- **用途**: 解密存储的 API 密钥
- **安全性**: 只有持有正确密钥才能解密
- **应用场景**: 在调用外部 API 时解密密钥

### 4. 密钥显示

#### get_api_key_prefix()
```python
def get_api_key_prefix(api_key: str, prefix_len: int = 8) -> str:
    """
    获取 API Key 前缀（用于显示）

    Args:
        api_key: 原始 API Key 字符串
        prefix_len: 前缀长度，默认 8 个字符

    Returns:
        str: API Key 前缀
    """
```
**源码实现**:
```python
def get_api_key_prefix(api_key: str, prefix_len: int = 8) -> str:
    """
    获取 API Key 前缀（用于显示）

    Args:
        api_key: 原始 API Key 字符串
        prefix_len: 前缀长度，默认 8 个字符

    Returns:
        str: API Key 前缀
    """
    if not api_key:
        return ""
    return api_key[:prefix_len] if len(api_key) > prefix_len else api_key
```

- **用途**: 安全地显示 API Key（只显示前几位）
- **特点**: 避免完整密钥泄露
- **应用**: 在用户界面中显示密钥时使用

## 使用示例

### 1. 加密存储 API 密钥
```python
from app.utils.encryption import encrypt_api_key

# 原始 API 密钥
original_api_key = "sk-1234567890abcdef"

# 加密存储
encrypted_key = encrypt_api_key(original_api_key)
print(f"加密后的密钥: {encrypted_key}")

# 存储到数据库或配置文件中
```

### 2. 使用时解密
```python
from app.utils.encryption import decrypt_api_key

# 从数据库获取加密的密钥
encrypted_key_from_db = "..."

# 解密使用
api_key = decrypt_api_key(encrypted_key_from_db)

# 调用外部 API
response = requests.get(api_url, headers={"Authorization": f"Bearer {api_key}"})
```

### 3. 安全显示
```python
from app.utils.encryption import get_api_key_prefix

api_key = "sk-1234567890abcdef"
prefix = get_api_key_prefix(api_key)
print(f"API Key: {prefix}...")  # 显示: sk-123456...
```

## 安全考虑

### 1. 密钥管理
- **生产环境**: 必须通过环境变量配置 `ENCRYPTION_KEY`
- **开发环境**: 自动生成临时密钥（但建议配置）
- **密钥轮换**: 支定期更换加密密钥

### 2. 加密算法
- **Fernet**: 基于 AES-128-CBC 和 HMAC
- **特性**: 
  - 提供认证加密
  - 防止篡改
  - 前向安全性

### 3. 存储安全
- 加密后的密钥可以安全存储在数据库或配置文件中
- 建议结合数据库加密存储
- 定期备份加密密钥
