# RAG（检索增强生成）

## 一句话定义

RAG（Retrieval-Augmented Generation）是在生成回答前，从外部知识库检索相关证据，并把证据与问题一起交给生成模型的系统架构。

## 它解决什么问题

- 模型训练知识存在截止时间。
- 模型可能生成缺少事实依据的内容。
- 通用模型缺少私有领域和个人资料。
- 参数化知识难以低成本、实时更新。

以上是本书给出的主要动机。[[RAG应用开发与实战手册-原文#2.1 什么是 RAG|来源]]

## 最小组成

1. 可被处理的外部文档。
2. 文档解析、切分和元数据。
3. 将问题与资料匹配的检索器。
4. 将检索结果组合成上下文的步骤。
5. 基于上下文生成回答的模型。
6. 能返回或记录来源的追溯机制。

## 容易误解

- RAG 不能保证消除幻觉；检索错误、上下文冲突和模型忽略证据仍会产生错误。
- 向量数据库不是 RAG 的唯一检索方式，还可以使用关键词、混合检索、知识图谱或结构化查询。
- RAG 不等于“把向量交给 LLM”。通常交给 LLM 的是检索到的文本片段。参见 [[RAG 基本工作流程]]。
- 一个能运行的 Demo 不等于效果可靠的系统；还需要评测集、检索指标、回答质量指标和可观测性。
